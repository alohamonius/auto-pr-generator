+## How is it useful?
+
+**CLI**
+
+&#x60;code2prompt&#x60; makes it easy to generate prompts for LLMs from your codebase. It traverses the directory, builds a tree structure, and collects information about each file. You can customize the prompt generation using Handlebars templates. The generated prompt is automatically copied to your clipboard and can also be saved to an output file. &#x60;code2prompt&#x60; helps streamline the process of creating LLM prompts for code analysis, generation, and other tasks.
+
+**SDK**
+
+&#x60;code2prompt&#x60; python binding helps programs to interact with codebase. It&#x27;s useful to ingest codebase for AI agent or automation scripts.

+## Features
+
+You can run this tool on the entire directory and it would generate a well-formatted Markdown prompt detailing the source tree structure, and all the code. You can then upload this document to either GPT or Claude models with higher context windows and ask it to:
+
+- Quickly generate LLM prompts from codebases of any size.
+- Customize prompt generation with Handlebars templates. (See the [default template](src/default_template.hbs))
+- Respects &#x60;.gitignore&#x60; (can be disabled with &#x60;--no-ignore&#x60;).
+- Filter and exclude files using glob patterns.
+- Control hidden file inclusion with &#x60;--hidden&#x60; flag.
+- Display the token count of the generated prompt. (See [Tokenizers](#tokenizers) for more details)
+- Optionally include Git diff output (staged files) in the generated prompt.
+- Automatically copy the generated prompt to the clipboard.
+- Save the generated prompt to an output file.
+- Exclude files and folders by name or path.
+- Add line numbers to source code blocks.
+
+You can customize the prompt template to achieve any of the desired use cases. It essentially traverses a codebase and creates a prompt with all source files combined. In short, it automates copy-pasting multiple source files into your prompt and formatting them along with letting you know how many tokens your code consumes.

+## Templates

+
+### [&#x60;document-the-code.hbs&#x60;](templates/document-the-code.hbs)
+

+
+### [&#x60;find-security-vulnerabilities.hbs&#x60;](templates/find-security-vulnerabilities.hbs)
+
+Use this template to generate prompts for finding potential security vulnerabilities in the codebase. It will look for common security issues and provide recommendations on how to fix or mitigate them.
+
+### [&#x60;clean-up-code.hbs&#x60;](templates/clean-up-code.hbs)
+
+Use this template to generate prompts for cleaning up and improving the code quality. It will look for opportunities to improve readability, adherence to best practices, efficiency, error handling, and more.
+
+### [&#x60;fix-bugs.hbs&#x60;](templates/fix-bugs.hbs)
+
+Use this template to generate prompts for fixing bugs in the codebase. It will help diagnose issues, provide fix suggestions, and update the code with proposed fixes.
+
+### [&#x60;write-github-pull-request.hbs&#x60;](templates/write-github-pull-request.hbs)
+
+Use this template to create GitHub pull request description in markdown by comparing the git diff and git log of two branches.
+
+### [&#x60;write-github-readme.hbs&#x60;](templates/write-github-readme.hbs)
+
+Use this template to generate a high-quality README file for the project, suitable for hosting on GitHub. It will analyze the codebase to understand its purpose and functionality, and generate the README content in Markdown format.
+
+### [&#x60;write-git-commit.hbs&#x60;](templates/write-git-commit.hbs)
+
+Use this template to generate git commits from the staged files in your git directory. It will analyze the codebase to understand its purpose and functionality, and generate the git commit message content in Markdown format.

+### [&#x60;improve-performance.hbs&#x60;](templates/improve-performance.hbs)

+## User Defined Variables
+
+&#x60;code2prompt&#x60; supports the use of user defined variables in the Handlebars templates. Any variables in the template that are not part of the default context (&#x60;absolute_code_path&#x60;, &#x60;source_tree&#x60;, &#x60;files&#x60;) will be treated as user defined variables.
+
+During prompt generation, &#x60;code2prompt&#x60; will prompt the user to enter values for these user defined variables. This allows for further customization of the generated prompts based on user input.
+
+For example, if your template includes &#x60;{{challenge_name}}&#x60; and &#x60;{{challenge_description}}&#x60;, you will be prompted to enter values for these variables when running &#x60;code2prompt&#x60;.
+
+This feature enables creating reusable templates that can be adapted to different scenarios based on user provided information.
+
+## Tokenizers
+
+Tokenization is implemented using [&#x60;tiktoken-rs&#x60;](https://github.com/zurawiki/tiktoken-rs). &#x60;tiktoken&#x60; supports these encodings used by OpenAI models:
+
+| Encoding name           | OpenAI models                                                             |
+| ----------------------- | ------------------------------------------------------------------------- |
+| &#x60;cl100k_base&#x60;           | ChatGPT models, &#x60;text-embedding-ada-002&#x60;                                  |
+| &#x60;p50k_base&#x60;             | Code models, &#x60;text-davinci-002&#x60;, &#x60;text-davinci-003&#x60;                       |
+| &#x60;p50k_edit&#x60;             | Use for edit models like &#x60;text-davinci-edit-001&#x60;, &#x60;code-davinci-edit-001&#x60; |
+| &#x60;r50k_base&#x60; (or &#x60;gpt2&#x60;) | GPT-3 models like &#x60;davinci&#x60;                                               |
+| &#x60;o200k_base&#x60;            | GPT-4o models                                                             |
+
+For more context on the different tokenizers, see the [OpenAI Cookbook](https://github.com/openai/openai-cookbook/blob/66b988407d8d13cad5060a881dc8c892141f2d5c/examples/How_to_count_tokens_with_tiktoken.ipynb)

+impl PrGenerator {
+    pub fn new(template_path: String, exclude_patterns: Vec&lt;String&gt;, claude_api_key: String, github_token: String) -&gt; Self {
+        Self {
+            template_path,
+            exclude_patterns,

+        }

+    fn get_current_branch(&amp;self) -&gt; Result&lt;String, Box&lt;dyn Error&gt;&gt; {
+        let output &#x3D; Command::new(&quot;git&quot;)
+            .args(&amp;[&quot;rev-parse&quot;, &quot;--abbrev-ref&quot;, &quot;HEAD&quot;])
+            .output()?;
+        
+        Ok(String::from_utf8(output.stdout)?.trim().to_string())
+    }
+
+    fn get_git_diff(&amp;self, base_branch: &amp;str, current_branch: &amp;str) -&gt; Result&lt;String, Box&lt;dyn Error&gt;&gt; {
+        let diff_arg &#x3D; format!(&quot;{}...{}&quot;, base_branch, current_branch);
+        let mut args &#x3D; vec![&quot;diff&quot;, &amp;diff_arg, &quot;--name-only&quot;];
+        
+        let output &#x3D; Command::new(&quot;git&quot;)
+            .args(&amp;args)
+            .output()?;
+            
+        Ok(String::from_utf8(output.stdout)?.trim().to_string())
+    }
+
+    fn get_git_log(&amp;self, base_branch: &amp;str, current_branch: &amp;str) -&gt; Result&lt;String, Box&lt;dyn Error&gt;&gt; {
+        let log_arg &#x3D; format!(&quot;{}...{}&quot;, base_branch, current_branch);
+        let output &#x3D; Command::new(&quot;git&quot;)
+            .args(&amp;[&quot;log&quot;, &amp;log_arg])
+            .output()?;
+            
+        Ok(String::from_utf8(output.stdout)?.trim().to_string())
+    }
+
+    async fn generate_code2prompt_output(&amp;self, changed_files: &amp;str) -&gt; Result&lt;String, Box&lt;dyn Error&gt;&gt; {
+        // Convert changed files to include patterns
+        let include_patterns: Vec&lt;String&gt; &#x3D; changed_files
+            .split(&#x27;\n&#x27;)
+            .filter(|f| !f.is_empty())
+            .map(|f| format!(&quot;**/{}&quot;, f))
+            .collect();
+
+        // Run code2prompt command
+        let include_arg &#x3D; include_patterns.join(&quot;,&quot;);
+        let output &#x3D; Command::new(&quot;code2prompt&quot;)
+            .args(&amp;[&quot;.&quot;, &quot;--include&quot;, &amp;include_arg, &quot;-t&quot;, &amp;self.template_path])
+            .output()?;
+
+        Ok(String::from_utf8(output.stdout)?.trim().to_string())
+    }
+
+    async fn get_claude_response(&amp;self, prompt: &amp;str) -&gt; Result&lt;String, Box&lt;dyn Error&gt;&gt; {
+        let client &#x3D; Client::new();
+        
+        let request &#x3D; ClaudeRequest {
+            model: &quot;claude-3-opus-20240229&quot;.to_string(),
+            max_tokens: 4000,
+            messages: vec![Message {
+                role: &quot;user&quot;.to_string(),
+                content: prompt.to_string(),
+            }],
+        };
+
+        let response &#x3D; client
+            .post(CLAUDE_API_ENDPOINT)
+            .header(&quot;x-api-key&quot;, &amp;self.claude_api_key)
+            .header(&quot;anthropic-version&quot;, &quot;2023-06-01&quot;)
+            .json(&amp;request)
+            .send()
+            .await?
+            .json::&lt;ClaudeResponse&gt;()
+            .await?;
+
+        Ok(response.content[0].text.clone())
+    }
+
+    pub async fn generate_pr_description(&amp;self, base_branch: &amp;str) -&gt; Result&lt;String, Box&lt;dyn Error&gt;&gt; {
+        let current_branch &#x3D; self.get_current_branch()?;
+        
+        // Get changed files
+        let changed_files &#x3D; self.get_git_diff(base_branch, &amp;current_branch)?;
+        
+        // Generate code2prompt output
+        let code_analysis &#x3D; self.generate_code2prompt_output(&amp;changed_files).await?;
+        
+        // Get git information
+        let git_log &#x3D; self.get_git_log(base_branch, &amp;current_branch)?;
+
+        // Prepare prompt for Claude
+        let prompt &#x3D; format!(
+            &quot;Please analyze this pull request and generate a comprehensive description.\n\n\
+            Code Analysis:\n{}\n\n\
+            Git Log:\n{}\n\n\
+            Please format the response as a well-structured markdown document with:\n\
+            1. A clear title\n\
+            2. Summary of changes\n\
+            3. Technical details\n\
+            4. Impact and considerations&quot;,
+            code_analysis, git_log
+        );
+
+        // Get response from Claude
+        let pr_description &#x3D; self.get_claude_response(&amp;prompt).await?;
+        
+        // Save to file
+        fs::write(&quot;pr_prompt.md&quot;, &amp;pr_description)?;
+        
+        Ok(pr_description)
+    }
+}
